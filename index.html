Техническое задание: Обновление системы обработки изображения (QR / орнамент) 🎯 Цель Добавить инструменты для точной настройки извлечения данных из изображения (в частности — QR-кода в синем канале), включая: загрузку статичных изображений (RAW/DNG), бинаризацию, гамма-коррекцию, подготовку под LUT (пресеты красок). 1. 📂 Модуль загрузки изображения Функционал: Добавить кнопку: «Загрузить файл» Реализовать Поддержка форматов: .jpg .png .dng (если возможно; иначе предусмотреть расширяемость) Логика: При загрузке файла: изображение заменяет видеопоток с камеры обработка применяется к загруженному изображению Возможность вернуться обратно к камере (опционально) 2. ⚫ Бинаризация (Thresholding) UI: Чекбокс: «Включить бинаризацию» Слайдер: «Порог (0–255)» отображается только если бинаризация включена Логика: if (gray > threshold) { output = 255; } else { output = 0; } Требования: Применяется после перевода изображения в grayscale Должна работать в реальном времени (если используется камера) Должна применяться к статичному изображению 3. 🌗 Гамма-коррекция (Gamma) UI: Слайдер: «Гамма» диапазон: 0.1 – 3.0 значение по умолчанию: 1.0 Логика: corrected = 255 * Math.pow((value / 255), gamma); Требования: Применяется до бинаризации Должна улучшать читаемость текстуры (особенно для акрила) Работает независимо от бинаризации (может быть включена всегда) 4. 🎨 LUT (Look-Up Table) UI: Выпадающий список: «LUT / Пресет красок» Примеры пресетов: Default Acrylic Oil High Contrast Custom Логика: LUT применяется к RGB каналам до всех остальных операций Возможность легко добавить новые LUT outputColor = LUT[inputColor]; 5. 🔁 Порядок обработки (Pipeline) Обработка изображения должна идти строго в следующем порядке: 1. Загрузка изображения / кадра 2. Применение LUT 3. Извлечение каналов (особенно синий) 4. Перевод в grayscale (если требуется) 5. Гамма-коррекция 6. Бинаризация (если включена) 7. Отображение результата 6. 🧠 Дополнительные требования Производительность: Обработка в реальном времени (для камеры) Без заметных лагов Расширяемость: Возможность добавления: новых LUT новых фильтров алгоритмов декуплизации (k1, k2) UX: Интуитивный интерфейс Все контролы должны быть реактивными (live update) 7. 💡 Ключевая идея (контекст для модели) Система используется для: извлечения скрытых данных (QR) из изображений, где информация закодирована в синем канале, а красный и зелёный содержат визуальный орнамент. Поэтому: 👉 критично сохранить и правильно обрабатывать канал B 👉 минимизировать межканальный шум 👉 дать пользователю точную настройку (gamma + threshold + LUT)
